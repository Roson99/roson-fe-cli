## git 提交规范:

所有的提交建议使用 标识：内容 的形式，说明此次提交的目的，使每次提交都有价值

```
标识        说明
feature	    新功能
fix	        修补 bug
docs	    文档（documentation）
style	    样式更改
format	    代码格式化不影响运行
refactor	重构（即不是新增功能，也不是修改 bug 的代码变动）
test	    增加测试
chore	    构建过程或辅助工具的变动
```
